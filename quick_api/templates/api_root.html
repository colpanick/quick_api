Documenatation

  • POST, PUT, and PATCH requests should include a Content-Type: application/json header. Otherwise, it will return a 415 status code and no changes will be made.
  • JSON "id" fields will be ignored in POST and PATCH requests

Routes

  • GET /api/dataset
  • GET /api/dataset/1
  • POST /api/dataset
  • PUT /api/dataset/1
  • PATCH /api/dataset/1
  • DELETE /api/dataset/1

Sort

  • GET /api/dataset?_sort=field&_order=asc_or_dsc
